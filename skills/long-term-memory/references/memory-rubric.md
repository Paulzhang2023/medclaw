# Memory Rubric

Use this rubric before saving or reusing a memory item.

## Keep If

- The fact will likely matter again in future sessions.
- The statement changes execution, not just background curiosity.
- The fact is specific enough to act on.
- The memory can be expressed without copying large chunks of dialogue.

## Discard If

- The fact is only useful for the current task.
- The statement is vague, flattering, or personality-theory rather than actionable guidance.
- The fact is likely to expire immediately.
- The item duplicates a stronger memory already stored.

## Preferred Categories

- User preference
- Writing preference
- Analysis preference
- Formatting preference
- Project convention
- Reusable lesson
- Background fact with practical impact

## Compression Rules

- Convert stories into rules.
- Convert repeated corrections into one stable instruction.
- Remove rhetorical framing.
- Keep one fact per line whenever possible.

## Examples

Good:

```text
Scope: user
Type: preference
Statement: User prefers Chinese responses for general discussion unless they request English.
Why it matters: Changes default reply language.
Confidence: high
```

```text
Scope: task-family:literary-writing
Type: preference
Statement: User prefers serious long-form fiction with stream-of-consciousness and dense psychological narration.
Why it matters: Guides style choices for future writing tasks.
Confidence: high
```

Bad:

```text
The user was very interested in a conversation about literature today and seemed emotional.
```

Reason: too interpretive and not operational.
